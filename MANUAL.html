1. Introdução

1.1 O que é o SGCD

O SGCD — Sistema de Gestão de Contratação Direta é uma aplicação web desenvolvida para organizar, acompanhar e documentar processos de Dispensa de Licitação conforme a Lei Federal nº 14.133/2021 (Nova Lei de Licitações e Contratos Administrativos).

O sistema funciona como uma aplicação local iniciada pelo arquivo Iniciar SGCD.bat, sem necessidade de instalação adicional. A partir da versão 2.0, os dados são armazenados em um banco de dados SQLite (sgcd.db) gerenciado por um servidor local, com suporte a múltiplos usuários autenticados simultaneamente.

O SGCD guia o agente de contratação por meio de um checklist com 18 etapas, cada uma com fundamento legal, orientações de preenchimento e possibilidade de anexar documentos. Ao final de cada etapa — ou a qualquer momento —, o sistema gera automaticamente os documentos necessários para o processo.

1.2 Para quem é destinado

O SGCD é destinado a agentes de contratação pública, pregoeiros, comissões de contratação e servidores responsáveis pela condução de processos de Dispensa de Licitação em órgãos e entidades da Administração Pública Federal, Estadual ou Municipal que adotam a Lei nº 14.133/2021.

O sistema é especialmente útil para:

  • Agentes que conduzem múltiplos processos simultaneamente e precisam de organização centralizada
  • Órgãos que buscam padronizar o fluxo de contratações diretas
  • Servidores que precisam gerar documentação padronizada com dados do processo
  • Equipes que necessitam controlar prazos e conformidade dos processos

1.3 Requisitos mínimos

O SGCD não exige instalação de software adicional. Os únicos requisitos são:

  • Navegador moderno: Google Chrome (versão 90 ou superior) ou Microsoft Edge (versão 90 ou superior) — o sistema detecta e abre automaticamente o navegador disponível
  • Python 3.7 ou superior: necessário para executar o servidor local (server.py). Instale em python.org e marque "Add Python to PATH" durante a instalação
  • JavaScript habilitado: necessário para o funcionamento da aplicação (habilitado por padrão em todos os navegadores modernos)
  • Conexão à internet: necessária apenas para consulta automática de CNPJ via ReceitaWS/BrasilAPI e para envio de e-mails via SMTP
Importante — Como abrir o sistema: O SGCD deve ser aberto exclusivamente pelo arquivo Iniciar SGCD.bat. Abrir o arquivo SGCD.html diretamente pelo navegador (clique duplo) cria uma base de dados separada e isolada, os dados não serão os mesmos, e o envio de e-mails ficará indisponível. Sempre use o Iniciar SGCD.bat.
Nota: Ao abrir o SGCD diretamente pelo arquivo HTML, o sistema exibe automaticamente uma tela de bloqueio informando o procedimento correto, evitando o uso acidental da base errada.

1.4 Armazenamento de dados

Importante: A partir da versão 2.0, todos os dados do SGCD (processos, fornecedores, configurações e anexos) são armazenados no arquivo sgcd.db (SQLite), localizado na mesma pasta do sistema. Os dados não dependem do navegador e não serão perdidos ao limpar o cache ou trocar de navegador.

O sistema opera em http://localhost:3000 (iniciado pelo Iniciar SGCD.bat). Nunca abra o arquivo SGCD.html diretamente pelo navegador — sem o servidor, o sistema não funciona.

Atenção: Faça backups regulares do arquivo sgcd.db ou use a função de exportação em Configurações → Backup de Dados. A perda ou exclusão do arquivo sgcd.db implica perda permanente dos dados.

2. Primeiro Acesso — Configuração Inicial

Antes de criar o primeiro processo, é essencial configurar o perfil do usuário. Essas informações serão utilizadas automaticamente em todos os documentos gerados pelo sistema (cabeçalhos, assinaturas, identificação do órgão etc.).

2.1 Acessando as configurações

Para acessar as configurações:

  1. Localize a barra lateral no lado esquerdo da tela
  2. Clique no ícone de engrenagem (⚙), geralmente posicionado na parte inferior da barra lateral
  3. O painel de Configurações será aberto com as abas disponíveis
Primeiro acesso: Ao abrir o SGCD pela primeira vez (sem perfil configurado), o sistema detecta automaticamente o primeiro uso e abre as Configurações com uma mensagem de boas-vindas, orientando o preenchimento dos dados antes de criar processos.

2.2 Campos do perfil

Preencha os seguintes campos na aba Perfil do Agente:

Dica: Preencha todos os campos antes de criar o primeiro processo. Uma vez configurados, esses dados são aplicados automaticamente em todos os documentos, economizando tempo e garantindo padronização.

2.3 Brasão personalizado

O sistema permite inserir o brasão ou logotipo institucional do órgão, que será exibido no cabeçalho de todos os documentos gerados. Para configurar:

  1. Na aba de Configurações, localize o campo "Brasão / Logotipo"
  2. Clique em "Escolher arquivo" e selecione a imagem do brasão (formato PNG, JPG ou SVG)
  3. O brasão será armazenado junto com as configurações e exibido nos documentos

Após preencher todos os campos, clique em "Salvar Configurações". Uma mensagem de confirmação será exibida.

3. Visão Geral da Interface

A interface do SGCD é dividida em três áreas principais: a barra superior (header), a barra lateral (sidebar) e a área de conteúdo central (dashboard ou processo aberto).

3.1 Barra superior (header)

A barra superior contém os seguintes elementos:

  • Nome do sistema (SGCD) e versão, no lado esquerdo
  • Campo de busca global: pesquisa em todos os campos de todos os processos cadastrados. Pode ser ativado pelo atalho Ctrl+K
  • Data atual: exibida no lado direito da barra superior
  • Indicadores de alerta: ícones que sinalizam processos com pendências urgentes

3.2 Barra lateral (sidebar)

A barra lateral (220px, fixa à esquerda) organiza a navegação em seções e exibe informações do usuário logado no rodapé:

  • Cabeçalho: exibe "SGCD", o subtítulo "Contratação Direta" e o nome do órgão configurado
  • Seção Principal — Dashboard: retorna ao dashboard de processos; exibe badge com o total de processos cadastrados
  • Seção Contratações — Fornecedores: acessa o cadastro de fornecedores
  • Seção Contratações — Agenda: abre a Agenda de Vencimentos; exibe badge vermelho com a contagem de alertas ativos
  • Seção Administração — Auditoria: abre a Trilha de Auditoria Global
  • Seção Administração — Notificações: abre o painel de alertas e pendências
  • Seção Administração — Configurações: abre as configurações do sistema
  • Rodapé do sidebar: exibe avatar com iniciais, nome e cargo do usuário logado; botão de Sair (volta à tela de login sem encerrar o servidor) e botão de Fechar Sistema (faz logout e encerra o servidor)

3.3 Dashboard

O dashboard é a tela principal do sistema, exibida ao abrir o SGCD. Ele apresenta:

  • Stat cards: cartões com resumo numérico — total de processos, em andamento, concluídos e não iniciados
  • Barra de filtros: permite filtrar a lista de processos por texto, status, unidade, período e faixa de valor
  • Lista de processos: exibe os processos cadastrados em formato de cards, tabela ou Kanban
  • Visualização Kanban: botão de alternância agrupa os processos em colunas por fase processual (Instrução, Publicado, Análise, Adjudicação, Contratação, Concluído). Os filtros ativos do dashboard são aplicados ao Kanban
  • Botão "+ Novo Processo": inicia o cadastro de um novo processo
  • Botão de Relatório: gera o relatório geral imprimível
  • Botão CSV: exporta os processos visíveis para arquivo CSV
Nota: Alertas automáticos aparecem no dashboard para processos parados há mais de 15 dias ou com prazos vencidos, destacados em vermelho ou amarelo para facilitar a identificação.

4. Cadastro de Fornecedores

O SGCD mantém um cadastro de fornecedores integrado com os processos. Fornecedores cadastrados podem ser vinculados às propostas e ao processo, facilitando o reaproveitamento de dados e o controle de certidões.

4.1 Consultando e cadastrando

Para acessar o cadastro de fornecedores, clique no ícone de prédio na barra lateral. A tela exibirá a lista de fornecedores cadastrados e um botão para adicionar novo.

Os fornecedores são exibidos como cards na lista. Clique em qualquer card para expandi-lo e visualizar todos os dados do fornecedor sem sair da tela. Clique novamente (ou em "Fechar") para recolher. Apenas um card pode estar aberto por vez.

Para cadastrar um novo fornecedor:

  1. Clique em "+ Novo Fornecedor"
  2. Digite o CNPJ do fornecedor no campo indicado
  3. Clique em "Consultar CNPJ" — o sistema buscará automaticamente os dados na ReceitaWS (fonte primária) ou BrasilAPI (fallback); requer conexão à internet
  4. Os campos serão preenchidos automaticamente: razão social, nome fantasia, endereço, atividade econômica (CNAE), situação cadastral, QSA e contato
  5. Verifique e complete os dados se necessário
  6. Clique em "Salvar"

Para editar um fornecedor já cadastrado, clique no botão "✏ Editar" no canto do card (sem precisar abrir modal). O formulário abre diretamente abaixo do card na lista. É possível também re-consultar o CNPJ na Receita Federal durante a edição para atualizar os dados.

Dica: Ao vincular fornecedores do cadastro às propostas (em vez de digitá-los manualmente), o histórico de participações do fornecedor em processos anteriores fica disponível automaticamente no card, na aba "Dados".

4.2 Certidões e regularidade fiscal

Para cada fornecedor, é possível registrar as certidões de regularidade fiscal, trabalhista e previdenciária. Com o card do fornecedor expandido, clique na aba "Certidões" e informe:

  • Tipo da certidão (ex.: FGTS, Receita Federal, INSS, Trabalhista, Municipal, Estadual)
  • Situação: Regular, Irregular ou Vencida
  • Data de emissão e data de vencimento

O formulário de certidão abre diretamente dentro do card, sem modal separado. Ao salvar ou cancelar, o sistema volta para a lista de certidões do fornecedor. O sistema exibe alertas quando certidões estão próximas do vencimento ou já vencidas, auxiliando na etapa de habilitação (Etapa 12).

4.3 Verificação de impedimento

No card do fornecedor (aba "Dados"), há links diretos para consulta nos portais de transparência:

  • CEIS — Cadastro de Empresas Inidôneas e Suspensas
  • CNEP — Cadastro Nacional de Empresas Punidas
  • Portal da Transparência — consulta geral de impedimentos

Clique nos links correspondentes para abrir os portais e realizar a consulta. Registre o resultado da verificação no campo de observações do fornecedor.

5. Criando um Novo Processo

Para iniciar um novo processo de Dispensa de Licitação, clique no botão "+ Novo Processo" no dashboard. Um formulário será exibido com os campos necessários para o cadastro inicial.

5.1 Campos do processo

Campo Obrigatório Descrição
Objeto Sim Descrição sucinta do bem, serviço ou obra a ser contratado. Exemplo: "Aquisição de material de expediente para o setor administrativo"
Número do PA Recomendado Número do Processo Administrativo. O sistema sugere uma numeração automática (editável)
Número da DL Recomendado Número da Dispensa de Licitação. Numeração sugerida automaticamente e editável

Após preencher os campos, clique em "Criar". O processo será cadastrado e o sistema abrirá automaticamente o checklist de etapas.

5.2 Natureza, categoria e subcategoria

Ao criar ou editar o processo, é possível classificar a contratação por:

  • Natureza: Bens ou Serviços
  • Categoria: tipo de bem ou serviço (ex.: Material de Consumo, Serviços de TI, Obras)
  • Subcategoria: classificação mais específica dentro da categoria

Essa classificação é utilizada pelo sistema para o controle de fracionamento: processos da mesma categoria e natureza têm seus valores somados automaticamente para verificação do limite do Art. 75.

6. Checklist das 18 etapas

O coração do SGCD é o checklist estruturado com as 18 etapas do processo de Dispensa de Licitação. Cada etapa apresenta o fundamento legal, uma orientação sobre o que deve ser feito e campos para registro. A seguir, a descrição completa de cada etapa:

Etapa 1
Documento de Formalização de Demanda (DFD)
Fundamento: Art. 12, §1º da Lei 14.133/2021

O que fazer: Solicitar à unidade requisitante o Documento de Formalização de Demanda (DFD), que deve conter a descrição da necessidade, a quantidade estimada do bem ou serviço e a justificativa da contratação. O DFD é o documento que origina formalmente o processo e deve ser assinado pelo responsável da unidade solicitante.

  • Receba ou elabore o DFD com o chefe da unidade requisitante
  • Verifique se a descrição da necessidade está clara e objetiva
  • Confirme se há justificativa para a quantidade solicitada
  • Anexe o DFD assinado ao processo nesta etapa
Etapa 2
Estudo Técnico Preliminar (ETP)
Fundamento: Art. 18, §2º da Lei 14.133/2021

O que fazer: Verificar se o objeto da contratação exige a elaboração de Estudo Técnico Preliminar. Para compras simples, padronizadas e de baixo valor, o ETP pode ser dispensado. Para contratações de maior complexidade, como serviços contínuos, tecnologia da informação ou obras, o ETP é obrigatório.

  • Avalie a complexidade e o valor da contratação
  • Se o ETP for dispensável, registre a justificativa no campo de observações
  • Se exigido, anexe o ETP elaborado pela área técnica competente
Nota: A dispensa do ETP deve ser fundamentada e registrada no processo, não sendo automática. Consulte as normas internas do órgão e orientações do TCU sobre os casos de obrigatoriedade.
Etapa 3
Termo de Referência (TR)
Fundamento: Art. 6º, XXIII da Lei 14.133/2021

O que fazer: O Termo de Referência é o documento que define com precisão o objeto da contratação. Deve conter especificações técnicas do objeto, critérios de aceitação, obrigações do contratado, prazo de entrega ou execução, local de entrega e qualquer outro requisito técnico relevante. O TR é elaborado pela área técnica ou requisitante e deve ser juntado ao processo.

  • Receba o TR da unidade técnica responsável
  • Verifique se as especificações são claras, objetivas e não direcionam para marca ou fornecedor específico
  • Confirme se os critérios de aceitação e prazo de entrega estão definidos
  • Anexe o TR aprovado ao processo nesta etapa
Etapa 4
Pesquisa de Preços / Estimativa do Valor
Fundamento: IN SEGES nº 65/2021 — mínimo de 3 cotações válidas

O que fazer: Registrar as cotações de mercado obtidas e calcular o valor de referência da contratação. A IN SEGES 65/2021 exige pelo menos 3 fontes válidas, que podem incluir: fornecedores do ramo, portais governamentais (Painel de Preços, BEC, BLL), notas fiscais de compras anteriores e tabelas referenciais de preço.

  • Clique em "+ Adicionar cotação" para cada fonte consultada
  • Preencha: fornecedor, CNPJ, valor cotado, data e situação (Válida ou Descartada)
  • Descarte cotações inexequíveis ou desatualizadas e justifique no campo de observações
  • Calcule e registre o valor de referência (geralmente a média das cotações válidas)
  • Guarde os comprovantes: e-mails, prints de portais, declarações de fornecedores
  • Anexe o Mapa de Preços ou planilha de apuração ao processo
Atenção: Cotações com preços inexequíveis (muito abaixo do mercado) ou superfaturados devem ser marcadas como "Descartada", com a justificativa registrada no campo de observações.
Etapa 5
Indicação de Dotação Orçamentária
Fundamento: Art. 12, VII da Lei 14.133/2021

O que fazer: Verificar e registrar a disponibilidade orçamentária para a despesa, confirmando junto ao setor de orçamento/finanças que há dotação suficiente para a contratação.

  • Preencha Programa/Ação, Elemento de Despesa, Fonte de Recursos e Dotação Completa
  • Confirme junto ao setor financeiro a disponibilidade de saldo orçamentário
  • Anexe a certidão ou declaração de disponibilidade orçamentária, se emitida pelo setor competente
Etapa 6
Enquadramento Legal
Fundamento: Art. 75 c/c Art. 72, III da Lei 14.133/2021

O que fazer: Fundamentar o enquadramento do processo na hipótese de Dispensa de Licitação cabível, com base nos incisos do Art. 75 da Lei 14.133/2021. A justificativa deve ser explícita e objetiva, demonstrando por que o caso se encaixa na hipótese invocada. Selecione também o Critério de Seleção que será usado no julgamento das propostas (menor preço, maior desconto, melhor técnica etc.), conforme o Art. 33 da Lei 14.133/2021.

As hipóteses mais comuns são:

  • Art. 75, I: obras e serviços de engenharia ou serviços de manutenção de veículos automotores até R$ 30.000,00
  • Art. 75, II: outros bens e serviços até R$ 65.492,11 (conforme Decreto nº 11.317/2022)
  • Art. 75, III: hipóteses de emergência ou calamidade pública
  • Art. 75, IV: contratação com órgãos ou entidades da Administração Pública
  • Incisos V a XVI: outras hipóteses específicas

Use o botão "Gerar Justificativa de Enquadramento Legal" para gerar automaticamente o documento exigido pelo Art. 72, III da Lei 14.133/2021, com o corpo jurídico declarando o enquadramento e a referência ao inciso aplicável.

Nota: Os valores-limite do Art. 75, I e II são atualizáveis por decreto. Consulte sempre a redação vigente. O critério de seleção padrão é "menor preço" — altere apenas se o TR exigir outro critério.
Etapa 7
Parecer do Controle Interno
Fundamento: Art. 8º da Lei 14.133/2021

O que fazer: Encaminhar o processo ao Controle Interno Municipal para análise prévia da regularidade e conformidade da contratação antes de seguir para a Procuradoria-Geral. O Controle Interno verifica a adequação orçamentária, o enquadramento legal e a regularidade formal do processo.

  • Encaminhe o processo instruído ao setor de Controle Interno
  • Aguarde a emissão do parecer e registre seu número no campo "Nº do Parecer do Controle Interno"
  • Digitalize e anexe o parecer assinado ao processo nesta etapa
Etapa 8
Parecer Jurídico da Procuradoria-Geral
Fundamento: Art. 53 da Lei 14.133/2021

O que fazer: Encaminhar o processo à Procuradoria-Geral (ou assessoria jurídica competente) para emissão de Parecer Jurídico sobre a legalidade e adequação da contratação. O parecer analisa o enquadramento legal, a regularidade do processo e pode apontar ressalvas ou condicionantes.

  • Encaminhe o processo instruído até esta etapa à assessoria jurídica
  • Aguarde a emissão do parecer e registre seu número no campo "Nº do Parecer Jurídico"
  • O número do parecer será referenciado automaticamente no despacho da Autorização de Abertura (Etapa 9)
  • Digitalize e anexe o parecer assinado ao processo nesta etapa
Nota: O número do Parecer Jurídico registrado nesta etapa é usado automaticamente no texto da Autorização de Abertura gerada na Etapa 9, garantindo a rastreabilidade documental do processo.
Etapa 9
Autorização da Autoridade Competente
Fundamento: Art. 72 da Lei 14.133/2021

O que fazer: Submeter o processo à autoridade competente (ordenador de despesa) para autorização formal do início do processo de Dispensa de Licitação. Use o botão "Gerar Autorização" para gerar o despacho preenchido com os dados do processo — incluindo, automaticamente, a referência ao Parecer Jurídico registrado na Etapa 8.

  • Clique em "Gerar Autorização" no painel da etapa
  • Imprima o documento ou salve como PDF
  • Submeta à assinatura da autoridade competente
  • Digitalize e anexe o documento assinado ao processo nesta etapa
Etapa 10
Elaboração e Publicação do Aviso de Dispensa
Fundamento: Art. 75, §3º e Art. 54, §1º da Lei 14.133/2021 — prazo mínimo 3 dias úteis

O que fazer: Elaborar o Aviso de Dispensa e publicá-lo no PNCP e no Diário Oficial. Use o botão "Gerar Aviso de Dispensa" para gerar o documento completo, preenchido com os dados do processo e o Critério de Seleção definido na Etapa 5. O aviso contém: identificação do órgão, objeto, valor estimado, critério de julgamento, condições de participação, seção sobre a proposta mais vantajosa (Art. 59), data limite e canais de divulgação do resultado.

  • Clique em "Gerar Aviso de Dispensa", revise, imprima ou salve como PDF e assine
  • Acesse o PNCP (www.pncp.gov.br) e publique o aviso completo
  • Publique também o aviso no Diário Oficial do Município (eletrônico ou impresso)
  • Registre no sistema: data de publicação, data de encerramento das propostas, link do PNCP e link do Diário Oficial
  • O sistema calcula automaticamente a data mínima de encerramento (3 dias úteis) e alerta se a data informada for inferior ao mínimo legal
  • Aguarde o prazo de 3 dias úteis antes de prosseguir
Atenção: A ausência de publicação ou o não cumprimento do prazo de 3 dias úteis é irregularidade grave que pode nulificar o processo.
Nota: O Aviso incluirá automaticamente os URLs do Site Oficial e Diário Oficial configurados nas Configurações. Configure esses campos em ⚙ Configurações antes de gerar o documento.
Etapa 11
Recebimento e Análise de Propostas
Fundamento: Art. 75, §3º da Lei 14.133/2021

O que fazer: Registrar as propostas recebidas dos fornecedores e realizar a análise comparativa. O SGCD possui uma seção específica para gerenciamento de propostas nesta etapa.

  • Para cada proposta recebida, clique em "+ Adicionar Proposta"
  • Preencha: fornecedor, CNPJ, valor proposto, data da proposta, validade e observações
  • Analise as propostas e marque a vencedora
  • Use o botão "Mapa de Preços" para gerar o documento comparativo

Veja a Seção 8 deste manual para detalhes completos sobre o gerenciamento de propostas.

Etapa 12
Habilitação do Fornecedor
Fundamento: Art. 62 a 70 da Lei 14.133/2021

O que fazer: Verificar a regularidade fiscal, trabalhista, previdenciária e jurídica do fornecedor vencedor. A habilitação é etapa obrigatória antes da adjudicação.

  • Consulte as certidões negativas de débitos (CND) do fornecedor vencedor
  • Verifique a regularidade: Receita Federal, FGTS, INSS, trabalhista, estadual e municipal
  • Consulte o CEIS e CNEP para verificar impedimentos
  • Registre as certidões no cadastro do fornecedor no sistema
  • Verifique se o fornecedor possui os documentos de habilitação jurídica (contrato social, CNPJ ativo)
Etapa 13
Justificativa da Escolha do Fornecedor
Fundamento: Art. 72, VI da Lei 14.133/2021

O que fazer: Registrar formalmente a razão da escolha do fornecedor contratado — exigência distinta da habilitação (que verifica apenas a regularidade fiscal e jurídica) e da justificativa de preço (já coberta pela Etapa 4). O Art. 72, VI exige que o processo de contratação direta seja instruído com a "razão da escolha do contratado", explicando por que aquele fornecedor específico foi selecionado (menor preço entre os habilitados, melhor técnica, exclusividade etc.).

  • Registre a justificativa no campo "Justificativa da Escolha do Fornecedor", referenciando os critérios objetivos que levaram à escolha
  • Relacione a justificativa ao Critério de Seleção definido na Etapa 6 e às propostas analisadas na Etapa 11
  • Use o botão "Gerar Justificativa da Escolha do Fornecedor" para gerar o documento formal, preenchido com os dados do processo, do fornecedor e do texto de justificativa registrado
  • Anexe documentos de apoio, se houver (ex.: comparativo de propostas, parecer técnico)
Nota: Etapa incluída a partir da v2.13.0, por sugestão do Controle Interno, para reforçar a instrução do processo conforme o Art. 72, VI da Lei 14.133/2021.
Etapa 14
Adjudicação
Fundamento: Art. 71 da Lei 14.133/2021

O que fazer: Declarar formalmente o vencedor do processo. A adjudicação é o ato pelo qual o agente de contratação declara o vencedor e o objeto da contratação a ele atribuído, após a habilitação.

  • Elabore o termo ou despacho de adjudicação com o nome do fornecedor vencedor, CNPJ e valor
  • Assine o documento como agente de contratação
  • Anexe o termo de adjudicação ao processo
  • Marque esta etapa como concluída no sistema
Etapa 15
Homologação
Fundamento: Art. 71 da Lei 14.133/2021

O que fazer: Submeter o processo à autoridade competente para homologação. A homologação é o ato da autoridade superior que ratifica o processo, conferindo-lhe validade. Após a homologação, o processo está apto para a emissão da Nota de Empenho e celebração do instrumento contratual.

  • Organize o processo com todos os documentos das etapas anteriores
  • Elabore despacho de encaminhamento para a autoridade competente
  • Submeta para assinatura da autoridade
  • Anexe o despacho de homologação assinado ao processo
Etapa 16
Nota de Empenho
Fundamento: Art. 58 da Lei 4.320/1964

O que fazer: Após a homologação, acionar o setor financeiro/orçamentário para emissão da Nota de Empenho (NE), que reserva o crédito orçamentário para a despesa. A NE é emitida no sistema de execução orçamentária do órgão (ex.: SIAFEM, SIAFEWEB, sistema municipal).

  • Encaminhe o processo ao setor financeiro com despacho solicitando emissão da NE
  • Informe: fornecedor, CNPJ, valor, dotação orçamentária, objeto e prazo
  • Receba a NE emitida e anexe ao processo
  • Registre o número da NE no campo correspondente no SGCD
Etapa 17
Instrumento Contratual
Fundamento: Art. 95 da Lei 14.133/2021

O que fazer: Formalizar o instrumento contratual com o fornecedor vencedor. A lei permite diferentes instrumentos conforme o valor e a natureza:

  • Contrato formal: obrigatório para contratações de maior valor ou duração
  • Carta-contrato, nota de empenho ou autorização de compra: dispensam o contrato formal nos casos previstos no Art. 95
  • Ata de Registro de Preços (ARP): quando aplicável

Para contratações por Dispensa de Licitação de pequeno valor, frequentemente é suficiente a Nota de Empenho acompanhada do Pedido de Compra ou Fornecimento.

  • Verifique o instrumento adequado para o valor e tipo de contratação
  • Elabore e assine o instrumento com o fornecedor
  • Anexe o instrumento assinado ao processo
Etapa 18
Publicação do Contrato no PNCP
Fundamento: Art. 94 da Lei 14.133/2021 — obrigatória em até 20 dias úteis

O que fazer: Publicar o instrumento contratual (contrato, ARP ou NE) no PNCP. O Art. 94 da Lei 14.133/2021 exige a publicação em até 20 dias úteis a partir da assinatura do instrumento.

  • Acesse o PNCP e registre o contrato na forma exigida
  • Anexe o arquivo do instrumento contratual no portal
  • Anote o número e a data de publicação
  • Registre no SGCD a data de publicação
  • Marque esta etapa como concluída — o processo estará finalizado
Atenção: O prazo de 20 dias úteis para publicação do contrato no PNCP é improrrogável. O descumprimento sujeita o agente responsável às sanções administrativas previstas na lei.

7. Trabalhando com o Processo

7.1 Expandir e recolher etapas

No painel do processo, cada etapa é exibida como um painel retrátil. Para expandir uma etapa e ver seus campos e opções, clique no cabeçalho da etapa. Para recolher, clique novamente no cabeçalho. Isso permite focar na etapa em que está trabalhando sem perder a visão geral do processo.

7.2 Status das etapas

Cada etapa pode ter um dos seguintes status:

  • Pendente: etapa ainda não iniciada (status inicial)
  • Concluída: etapa finalizada com sucesso
  • Bloqueada: etapa que não pode ser avançada por alguma razão (ex.: pendência em etapa anterior, prazo de publicação aguardando)

Para alterar o status, clique no ícone de verificação (✔) no cabeçalho da etapa. Cada clique avança pelo ciclo: Pendente → Concluída → Bloqueada → Pendente. A barra de progresso do processo é atualizada automaticamente.

7.3 Anexar documentos

Cada etapa possui uma área para anexo de documentos. Para anexar:

  1. Expanda a etapa desejada
  2. Localize a seção "Documentos" ou o botão de clipe/anexo
  3. Clique em "Anexar Arquivo" e selecione o arquivo no computador
  4. Formatos aceitos: PDF, DOCX, DOC, PNG, JPG
  5. O arquivo será armazenado no IndexedDB junto com o processo
Atenção: Arquivos anexados ficam armazenados no navegador. Arquivos muito grandes podem afetar o desempenho do sistema. Recomenda-se compactar PDFs antes de anexar.

7.4 Editar informações do processo

Para editar as informações gerais do processo (objeto, valor, fundamentação, unidade etc.), localize o painel lateral direito quando o processo está aberto, ou clique no botão "Editar" no cabeçalho do processo. Altere os campos desejados e clique em "Salvar".

7.5 Reordenar etapas

Embora as 18 etapas sigam uma ordem lógica sugerida, é possível reordená-las conforme as necessidades específicas do processo. Para reordenar:

  1. Localize a alça de arrastar ⠿ no lado esquerdo do cabeçalho da etapa
  2. Clique e mantenha pressionado sobre a alça
  3. Arraste a etapa para a posição desejada
  4. Solte para confirmar a nova posição
Nota: A reordenação altera apenas a visualização no sistema e não implica alteração da ordem legal das atividades do processo. O agente de contratação é responsável por garantir a conformidade jurídica da sequência adotada.

7.6 Histórico e log de auditoria

O SGCD registra automaticamente todas as alterações realizadas no processo: mudanças de status de etapas, edições de campos, adição de propostas e anexos. Para visualizar o histórico do processo:

  1. No painel do processo, localize o botão "Histórico" no cabeçalho
  2. O log exibe cada alteração com: campo alterado, valor anterior, novo valor, usuário (configurado) e data/hora

O log de auditoria é somente leitura e não pode ser editado, garantindo a rastreabilidade das ações no processo.

Nota: O histórico exibido aqui é relativo ao processo aberto. Para visualizar o registro consolidado de todas as ações de todos os processos e fornecedores, acesse a Trilha de Auditoria Global pelo ícone na barra lateral (ver Seção 15).

7.7 Responsável preenchido automaticamente

Ao abrir um processo, o SGCD verifica se o campo Responsável das etapas está preenchido. Caso alguma etapa esteja sem responsável, o sistema preenche automaticamente com o nome do Agente de Contratação configurado nas Configurações do sistema.

Isso garante que nenhum processo fique com o campo de responsável vazio, sem exigir ação manual do usuário. O campo pode ser editado a qualquer momento dentro da etapa.

Dica: Configure o nome do agente de contratação em Configurações → Perfil antes de criar os primeiros processos para que o preenchimento automático funcione desde o início.

8. Gerenciamento de Propostas (Etapa 11)

A Etapa 11 conta com uma seção dedicada ao gerenciamento de propostas recebidas, que permite registrar, comparar e selecionar a proposta vencedora de forma organizada e com documentação automática.

8.1 Situações de proposta

Como adicionar uma proposta

  1. Expanda a Etapa 11 no checklist do processo
  2. Clique em "+ Adicionar Proposta"
  3. Preencha os campos:
CampoDescrição
FornecedorNome ou razão social do fornecedor que enviou a proposta
CNPJCNPJ do fornecedor (ou CPF, para pessoa física)
Valor PropostoValor total da proposta em reais
Data da PropostaData em que a proposta foi recebida
ValidadeData até quando a proposta é válida
SituaçãoEm análise, Aceita, Vencedora ou Recusada
ObservaçõesCampo livre para anotações (ex.: motivo de recusa)

Para vincular o fornecedor ao cadastro do sistema, clique no botão 📋 ao lado do campo de fornecedor e selecione o fornecedor já cadastrado. Isso popula automaticamente nome e CNPJ e vincula o histórico.

Definindo o vencedor

Ao alterar a situação de uma proposta para "Vencedora":

  • O sistema marca automaticamente as demais propostas como "Aceita"
  • O fornecedor vencedor é vinculado ao processo
  • Os dados do vencedor são utilizados nos documentos gerados (Ata de Sessão, Autorização de Abertura)
Nota: Apenas uma proposta pode ter a situação "Vencedora". Ao marcar uma nova proposta como vencedora, a anterior é automaticamente reclassificada.

Mapa de Preços

O Mapa de Preços é um documento comparativo das propostas recebidas, exigido pela IN SEGES 65/2021. Para gerá-lo:

  1. Registre todas as propostas na Etapa 11
  2. Clique em "Gerar Mapa de Preços"
  3. O documento será aberto em nova janela, pronto para impressão ou salvamento como PDF

O Mapa de Preços contém: cabeçalho com dados do processo, tabela comparativa com todos os fornecedores e valores, identificação do vencedor e o menor preço, data de geração e código de autenticidade.

8.2 Despacho de Recusa / Desclassificação

Quando uma proposta é marcada como "Recusada", o sistema exibe automaticamente o botão "📄 Despacho de Recusa" no card da proposta. Esse documento formaliza a decisão de recusa ou desclassificação da proposta, com fundamentação jurídica completa.

O despacho é gerado com base nos dados cadastrados nas Configurações (nome, cargo e matrícula do agente de contratação) e contém:

  • Identificação do processo (PA e DL)
  • Relação das propostas recusadas com fornecedor, CNPJ e valor
  • Decisão formal de recusa e desclassificação
  • Fundamentação: Art. 59 da Lei 14.133/2021 e Art. 50 da Lei 9.784/1999
  • Seção "Do Direito de Manifestação" com prazo de 10 dias úteis
  • Espaço para assinatura do agente de contratação
  • Código de autenticidade no rodapé
Dica: Preencha o campo "Observações" da proposta com o motivo técnico da recusa (ex.: "proposta acima do valor de referência", "documentação incompleta"). O motivo é incorporado automaticamente ao despacho.

8B. Habilitação do Fornecedor (Etapa 12)

A Etapa 12 trata da habilitação jurídica, fiscal, trabalhista e técnica do fornecedor vencedor, conforme Art. 69 da Lei Federal nº 14.133/2021. O sistema oferece uma lista estruturada de 21 certidões e consultas oficiais para controle completo da regularidade do fornecedor.

8B.1 Certidões e consultas oficiais

As certidões estão organizadas em três grupos:

GrupoItens
Cadastros e Consultas OficiaisCNPJ, QSA, SICAF (CRC e restrições), TCU (consolidada e inidôneos), CGU, CNJ, TCESP (licitação e repasse), Portal da Transparência
Certidões Fiscais e TrabalhistasCND Federal (RFB/PGFN), CND Estadual (PGE/SP), CND Municipal, CRF-FGTS, CNDT (TST), Certidão Simples Nacional, Certidão de Falência e Concordata
EngenhariaSUSEP — Certidão de Licenciamento, SUSEP — Apólice de Seguro Garantia (somente quando aplicável)

Para cada certidão é possível registrar:

  • Status: Pendente / Obtida / N/A (não aplicável)
  • Data de emissão e Data de validade (para certidões com status "Obtida")
  • Arquivo anexado (PDF ou imagem da certidão)

O botão "↗ Acessar" abre diretamente o portal oficial da certidão no navegador.

O sistema exibe alertas visuais de validade diretamente na lista de certidões:

  • Borda vermelha + badge "⚠ Vencida" — a data de validade já passou
  • Borda amarela + badge "⚠ Vence em breve" — a certidão vence nos próximos 30 dias

Esses alertas ajudam a identificar certidões que precisam ser renovadas antes da formalização do contrato.

8B.2 Sincronização com o cadastro de fornecedores

As datas de emissão e validade preenchidas na Etapa 12 são automaticamente sincronizadas com o cadastro do fornecedor vinculado ao processo. Certidões sincronizáveis são identificadas pelo indicador "⇄ fornecedor" ao lado dos campos de data.

A sincronização funciona nos dois sentidos:

  • Ao preencher datas na Etapa 12, o cadastro do fornecedor é atualizado
  • Ao abrir a Etapa 12 com um fornecedor vinculado, datas já cadastradas no fornecedor são pré-preenchidas automaticamente
Nota: Apenas certidões com prazo de validade definido são sincronizadas (ex.: CND Federal, FGTS, CNDT). Consultas sem validade fixa (CNPJ, QSA, TCU) não geram entrada no cadastro do fornecedor.

8B.3 Despacho de Inabilitação

Caso o fornecedor não comprove a regularidade exigida, o botão "📄 Despacho de Inabilitação" gera o documento formal de inabilitação. O sistema identifica automaticamente as certidões com status "Pendente" e as lista como motivação da decisão.

O documento contém:

  • Identificação do processo e do fornecedor (razão social e CNPJ)
  • Relação das certidões pendentes que motivam a inabilitação
  • Fundamentação: Art. 69 e Art. 72 da Lei 14.133/2021 e Art. 50 da Lei 9.784/1999
  • Seção "Do Direito de Manifestação" com prazo de 10 dias úteis
  • Espaço para assinatura do agente de contratação
  • Código de autenticidade no rodapé
Atenção: Certifique-se de que todas as certidões foram analisadas antes de gerar o despacho. Certidões marcadas como "N/A" não aparecem na lista de pendências.

10. Verificação de Conformidade

A funcionalidade de Verificação de Conformidade analisa automaticamente o processo e identifica pendências, inconsistências e itens que precisam de atenção antes da continuidade ou encerramento do processo.

Como usar

  1. Com o processo aberto, clique no botão "✔ Conformidade" no cabeçalho
  2. O sistema realizará a verificação automática e exibirá o resultado em uma lista

O que é verificado

  • Objeto: campo preenchido e com descrição suficiente
  • Fundamentação legal: inciso do Art. 75 informado
  • Valor estimado: valor preenchido e dentro dos limites da hipótese de dispensa
  • Unidade solicitante: campo preenchido
  • Número do processo: PA e DL informados
  • Fornecedor vencedor: vinculado ao processo
  • Propostas: mínimo de 3 propostas registradas (conforme IN SEGES 65/2021)
  • Certidões: regularidade fiscal do fornecedor vencedor
  • Checklist: etapas obrigatórias concluídas

O resultado exibe cada item como conforme (em verde) ou não conforme / pendente (em vermelho ou amarelo), com uma orientação sobre o que deve ser corrigido ou completado.

12. Backup e Restauração

Como os dados do SGCD ficam armazenados exclusivamente no navegador, é fundamental realizar backups regulares para evitar perda de informações.

Como fazer backup

  1. Clique no ícone de engrenagem (⚙) para abrir as Configurações
  2. Localize a seção "Backup de Dados"
  3. Clique em "Exportar Backup"
  4. Um arquivo JSON será baixado automaticamente pelo navegador com o nome sgcd-backup-[data].json
  5. Salve o arquivo em local seguro (pasta local, nuvem, pen drive)
Dica: Faça backup ao menos uma vez por semana, ou sempre antes de limpar o cache do navegador, atualizar o sistema ou formatar o computador.

Como restaurar um backup

  1. Nas Configurações, seção "Backup de Dados", clique em "Restaurar Backup"
  2. Selecione o arquivo JSON do backup salvo anteriormente
  3. Confirme a operação — todos os dados atuais serão substituídos pelos dados do backup
Atenção: A restauração substitui completamente todos os dados atuais (processos, fornecedores e configurações) pelos dados do arquivo de backup. Essa operação não pode ser desfeita. Faça um backup do estado atual antes de restaurar, se necessário.

Sincronizar com outro agente (mesclar backups)

Quando dois agentes usam o sistema em máquinas diferentes, a Restauração não é a ferramenta certa — ela apaga tudo e coloca só os dados do arquivo. Para juntar o trabalho dos dois sem perder nada, use Sincronizar backup, em Configurações → Backup de Dados:

  1. No computador do Agente 1, clique em "Exportar backup" (Sistema, JSON) e envie o arquivo ao Agente 2
  2. No computador do Agente 2, em "Sincronizar com outro agente", clique em "Sincronizar backup" e selecione o arquivo recebido
  3. Processos e fornecedores que só existem no arquivo são somados automaticamente
  4. Se algum processo ou fornecedor existir nos dois lados com alterações diferentes, uma tela de revisão mostra as duas versões lado a lado — marque quais devem vir do backup; os desmarcados mantêm a versão já existente na máquina
  5. Um backup de segurança do banco atual é criado automaticamente antes de aplicar qualquer alteração
Nota: Se, por coincidência, as duas máquinas tiverem criado processos com o mesmo número oficial (ex.: "PA 5/2026") para assuntos diferentes, o sistema avisa ao final da sincronização — nesse caso, renumeie manualmente um dos processos, já que os números de referência precisam ser únicos.

Auto-backup automático

O sistema realiza um auto-backup automático no localStorage sempre que um processo é salvo. Essa cópia serve como proteção emergencial contra perda acidental de dados, mas não substitui o backup em arquivo JSON. O auto-backup respeita limites de tamanho: se a base de dados ultrapassar 4 MB, o sistema exibe um aviso e não tenta salvar, evitando erros de quota do navegador.

Factory Reset — Limpeza total de dados

A função Factory Reset apaga permanentemente todos os dados do sistema: processos, fornecedores, certidões, configurações e brasão. Para evitar exclusões acidentais, o processo exige três confirmações sucessivas:

  1. Aviso inicial com descrição do que será apagado
  2. Confirmação de segurança digitando o nome do órgão configurado
  3. Confirmação final irrevogável
Atenção: O Factory Reset não pode ser desfeito. Faça um backup completo antes de executar esta operação. A função está disponível em Configurações → Backup de Dados → Limpar todos os dados.

13. Alertas de Fracionamento

O fracionamento de despesas é uma prática vedada pela Lei 14.133/2021. Consiste em dividir artificialmente uma contratação em parcelas menores para se enquadrar nos limites de Dispensa de Licitação e fugir de modalidades mais rigorosas. O SGCD possui um mecanismo automático de detecção de fracionamento.

Como funciona

O sistema monitora os processos cadastrados e:

  • Detecta processos com objetos similares (mesma categoria e subcategoria)
  • Calcula a soma dos valores estimados no exercício fiscal ou na janela temporal configurada
  • Emite um alerta quando a soma se aproxima ou ultrapassa os limites do Art. 75

Configurar a janela temporal

Em Configurações, é possível definir o período para o cálculo de fracionamento:

  • Ano civil: considera todos os processos do exercício fiscal (1º de janeiro a 31 de dezembro)
  • Período personalizado: define uma janela em meses (ex.: últimos 12 meses)
Nota: O alerta de fracionamento é uma ferramenta de apoio ao agente de contratação. A análise jurídica de configuração de fracionamento deve sempre ser realizada pelo agente e pela assessoria jurídica do órgão, levando em conta as especificidades de cada caso.

14. Dicas e Boas Práticas

As orientações abaixo ajudam a usar o SGCD de forma eficiente e segura:

Abra sempre pelo Iniciar SGCD.bat: nunca abra o arquivo SGCD.html diretamente pelo Windows Explorer ou pelo navegador. O BAT inicia o servidor local que permite o envio de e-mails e garante que você esteja trabalhando na base de dados correta. Abrir diretamente cria uma base de dados isolada e invisível ao sistema.
Backup antes de limpar o cache: os dados do SGCD ficam no navegador. Antes de limpar o histórico ou cache, sempre exporte um backup em Configurações → Backup de Dados.
Use sempre o mesmo navegador: os dados ficam vinculados ao perfil do navegador. Se você usar Chrome no trabalho, use sempre o Chrome para acessar o SGCD. Não misture navegadores (Chrome e Edge não compartilham dados).
Preencha PA e DL logo ao criar o processo: mesmo que provisórios, os números de PA e DL facilitam a rastreabilidade desde o início e são usados nos documentos gerados. Você pode editá-los posteriormente.
Vincule fornecedores do cadastro: em vez de digitar nome e CNPJ na proposta manualmente, use o botão 📋 para vincular fornecedores já cadastrados. Isso mantém o histórico de participações e aproveita os dados já registrados (certidões, endereço etc.).
Registre as propostas com data exata: a data de cada proposta é importante para demonstrar a conformidade da pesquisa de preços com a IN SEGES 65/2021. Propostas muito antigas podem ser desconsideradas.
Use a Verificação de Conformidade antes de gerar documentos: o botão "✔ Conformidade" aponta pendências que podem fazer os documentos saírem incompletos. Corrija-as antes de gerar a Ata ou a Autorização.
Mantenha o checklist atualizado: marque as etapas como concluídas à medida que avança. Isso garante que a barra de progresso e os alertas do dashboard reflitam a situação real do processo.
Registre observações nas etapas: cada etapa tem um campo de observações. Use-o para registrar informações relevantes: quem enviou o DFD, qual foi o resultado da consulta ao CEIS, por que uma proposta foi recusada etc. Isso documenta o processo e protege o agente de contratação em eventuais auditorias.

22. Backup Automático e Proteção de Dados

A partir da versão 1.18.0, o SGCD protege os dados do usuário de forma automática, sem necessidade de ação manual a cada alteração.

22.1 Snapshot automático

Toda vez que um processo é salvo, o sistema aguarda 3 segundos e grava automaticamente um snapshot compactado no armazenamento local do navegador (localStorage). Esse snapshot contém todos os processos e fornecedores cadastrados, exceto anexos de arquivo (para não exceder o limite de espaço do navegador).

Limite de tamanho: O localStorage comporta aproximadamente 5 MB. Se a base de dados exceder esse limite, o sistema exibirá um aviso solicitando a exportação de um backup manual.

O snapshot automático também é gravado quando o usuário fecha ou recarrega a aba do navegador, desde que a opção "Snapshot ao fechar" esteja ativa nas Configurações → Dados.

22.2 Banner de alerta de backup

Uma faixa de alerta laranja é exibida na parte inferior da tela quando:

  • Nenhum backup manual foi exportado ainda (primeiro uso do sistema); ou
  • O último backup manual foi exportado há mais de 7 dias.

O banner exibe o número de dias sem backup e um botão de exportação rápida. Ao exportar o backup, o banner é dispensado automaticamente. Clicar em "Lembrar depois" oculta o banner até o próximo carregamento.

22.3 Exportação manual recomendada

O backup manual (Configurações → Dados → Exportar Backup) gera um arquivo .json completo com processos, fornecedores, arquivos anexados, configurações e brasão. Este é o único formato que inclui anexos e que pode ser importado para restaurar o sistema em outro computador ou navegador.

Atenção: O snapshot automático é um mecanismo de proteção contra falhas imediatas — não substitui o backup manual. Guarde os arquivos de backup exportados em local seguro (pasta de rede, nuvem ou mídia externa).

  • Clicar em "Voltar e preencher" cancela a ação e retorna ao formulário da etapa.

23. Controle de Acesso por Senha

O SGCD permite configurar uma senha de abertura para proteger o acesso ao sistema em computadores compartilhados.

23.1 Configurando a senha

Acesse Configurações → Segurança. O painel exibe três campos:

  1. Senha atual — deixe em branco se ainda não há senha configurada.
  2. Nova senha — mínimo de 4 caracteres.
  3. Confirmar nova senha — deve ser idêntica ao campo anterior.

Clique em "Definir Senha" para salvar. Para remover a proteção, informe a senha atual e clique em "Remover Senha".

23.2 Tela de acesso restrito

Quando uma senha está configurada, o sistema exibe uma tela de acesso ao ser carregado. Essa tela apresenta:

  • O nome do órgão configurado nas Configurações → Organização.
  • A data do último backup exportado.
  • Um campo de senha com opção de visualizar os caracteres digitados.

Em caso de senha incorreta, o formulário exibe uma mensagem de erro com animação visual e limpa o campo para nova tentativa. Pressionar Enter no campo de senha aciona o botão de entrar.

Limitação técnica: A senha é armazenada como hash SHA-256 no localStorage do navegador. Essa proteção impede acesso acidental, mas não é equivalente a criptografia do banco de dados. Um administrador com acesso às ferramentas de desenvolvedor do navegador pode, em tese, limpar o hash e remover a proteção. Para ambientes que exijam segurança elevada, combine o uso do SGCD com senha de login do sistema operacional.

23.3 Recuperação de acesso

Se a senha for esquecida, é possível removê-la manualmente pelo console do navegador:

  1. Abra as ferramentas de desenvolvedor (F12).
  2. Acesse a aba Console.
  3. Execute: localStorage.removeItem('sgcd-access-hash')
  4. Recarregue a página.
Atenção: Este procedimento deve ser realizado apenas pelo administrador do sistema. Mantenha a senha anotada em local seguro.

24. Painel de Diagnóstico

O Painel de Diagnóstico analisa automaticamente todos os processos e fornecedores cadastrados, identificando inconsistências, campos obrigatórios faltantes e situações de não conformidade.

24.1 Como acessar

Acesse Configurações → Diagnóstico e clique em "Reanalisar". O sistema percorre todos os registros do banco de dados e apresenta o resultado em segundos.

24.2 O que é verificado

O diagnóstico agrupa os resultados em três categorias:

CategoriaCorSignificado
Erros críticosVermelhoSituações que comprometem a validade jurídica do processo ou a integridade dos dados
AvisosAmareloInconsistências que podem causar problemas, mas não necessariamente invalidam o processo
InformaçõesAzulCampos recomendados não preenchidos ou configurações incompletas

24.3 Verificações realizadas

Para cada processo, o diagnóstico verifica:

  • Processo concluído sem fornecedor vencedor identificado.
  • Processo concluído sem base legal (Art. 75, inciso) selecionada.
  • Processo com status "concluído" mas etapa de Homologação não marcada como concluída.
  • Aviso de Dispensa concluído sem data de publicação registrada.
  • Processo sem valor estimado informado.
  • Habilitação concluída sem fornecedor referenciado.
  • Processo sem objeto/descrição preenchido.
  • Processo em andamento sem número da Dispensa de Licitação.

Para o cadastro de fornecedores, o diagnóstico verifica:

  • CNPJs duplicados — dois ou mais fornecedores com o mesmo CNPJ cadastrado.
  • CNPJs inválidos — dígitos verificadores incorretos segundo o algoritmo da Receita Federal.

Para as configurações do sistema, o diagnóstico verifica:

  • CNPJ do órgão não informado (necessário para a exportação PNCP).
  • Código IBGE do município não informado (necessário para a exportação PNCP).
Boas práticas: Execute o diagnóstico antes de encaminhar um processo para publicação ou assinatura, e ao final de cada exercício financeiro para identificar processos com dados incompletos.
